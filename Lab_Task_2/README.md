<h1> AI Grid Simulation: A* and UCS Algorithms </h1>
This project implements a grid-based simulation using two AI pathfinding algorithms: A* and UCS (Uniform Cost Search). It simulates an agent navigating a grid to complete tasks, considering barriers that may obstruct its path.

<h2>Features</h2>
<ul>
  <li><b>A* Algorithm:</b> Uses a heuristic (Manhattan distance) to find the optimal path to the nearest task.</li>
  <li><b>UCS (Uniform Cost Search):</b> A search algorithm that finds the shortest path to the nearest task by considering the cost of each move.</li>
  <li><b>Dynamic Task Completion:</b> Tasks are dynamically generated on the grid, and the agent completes them by navigating around barriers.</li>
  <li><b>Pygame Visualization</b> Visual feedback via Pygame that shows grid, tasks, barriers, and the agent's movement (up, down, left, right).</li></ul>
<h2>Requirements</h2>
<ul><li>Python: 3.6 or higher</li>
<li>Pygame: For graphical display and user interaction</li></ul>
You can install the required libraries using <code>pip</code>:<br>
<code>pip install pygame</code>
<h2>How to Run</h2>
<ol><li>Clone the repository or download the files.</li>
<li>Run the run.py file to start the simulation.<br>
<code>python run.py</code></li></ol>
This will open a Pygame window where the agent navigates the grid to complete tasks using either the A* or UCS algorithm, which can be triggered by clicking on their respective buttons in the status panel.

<h2>Key Components</h2>
<h3><code>agent.py</code></h3>
<ul><li>Defines the Agent class, which represents the agent navigating the grid.</li>
<li>Implements methods for pathfinding using both A* and UCS.</li>
<li>Handles task completion and cost tracking.</li>
<li>Provides methods to move the agent along the path generated by each algorithm.</li></ul>

<h3><code>environment.py</code></h3>
<ul><li>Defines the Environment class, which manages the grid, tasks, and barriers.</li>
<li>Generates random task locations and barriers.</li>
<li>Ensures the agent's movement stays within the grid bounds and avoids barriers.</li></ul>

<h3><code>run.py</code></h3>
<ul>
  <li>Sets up the Pygame window and controls the main simulation loop.</li>
  <li>Draws the grid, agent, tasks, barriers, and displays the current state and costs.</li>
  <li>Handles user interactions (button clicks to start A* or UCS simulations).</li>
  <li>Updates the agent’s movement and task completion in real-time.</li>
</ul>

<h2>Simulation Workflow</h2>
<ol>
  <li><b>Grid Setup:</b> A grid is created with random tasks and barriers placed within it</li>
  <li><b>Agent Movement:</b>
    <ul>
      <li>The agent moves along the grid using the selected algorithm (A* or UCS).</li>
      <li>The agent completes tasks by reaching the task locations.</li>
      <li>Task completion is tracked, and the total cost of each path is displayed in real-time.</li>
    </ul>
  </li>
  <li><b>User Interaction:</b>
    <ul>
      <li>Two buttons, "A* Search" and "UCS Search," allow the user to start the simulation with the respective algorithm.</li>
      <li>The agent moves automatically, and users can observe the completed tasks, total path cost, and agent’s position in the status panel.</li>
    </ul>
  </li>
</ol>

<h2>Customization</h2>
<ul>
  <li>You can modify the grid size, number of tasks, and barriers by editing the constants in the <code>run.py</code> file.</li>
  <li>The agent's behavior can be tweaked in <code>agent.py</code>, such as adjusting the pathfinding algorithms or task completion logic.</li>
</ul>
